== WHAT THIS PROJECT IS ==

sw-cor24-monitor is a resident monitor and terminal controller for the
COR24 soft CPU. It boots at address 0, owns the UART hardware
exclusively, and provides a synchronous call/return program invocation
environment with structured argument passing and return codes.

Key design decisions already locked in (from research.txt):
- Monitor at address 0, runtime/service vector at 0x500, shell at
  0x1000, user programs at 0x2000+
- Only the monitor touches UART MMIO — all programs use service calls
- Synchronous call/return execution model (no scheduler/multitasking)
- Service vector table: putchar, getchar, write, readline, exit
- Invocation block: argc, argv, cmdline, flags passed to programs
- Program entry: int prog_main(mon_context *ctx) where ctx holds
  both invocation block and service vector pointers
- Return trampoline so monitor always regains control
- Program registry maps names to fixed entry addresses
- RC propagation: program RC flows back to shell as $rc
- Three error classes: program RC, monitor invocation failure,
  program fault
- Assembler for bootstrap/trampoline, C for policy/registry/shell
- Single foreground console owner model
- Future: terminal multiplexor for multi-CPU FPGA/MCU systems

== CONSTRAINTS ==

- tc24r C subset: no structs, no malloc/free, no string library,
  no varargs, single translation unit, 24-bit int, 8-bit char
- All data structures must use parallel arrays or packed flat memory
- UART I/O only (memory-mapped)
- Assembler for low-level bootstrap and trampoline code
- Keep static allocation limits reasonable
- Educational clarity over cleverness
